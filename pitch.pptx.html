ActMap Voice
Pre-speech decision layer

Route before
the agent speaks.

Voice agents should not retrieve documents for every request, and they should not speak from memory when a turn depends on policy, account state, or risk. ActMap reads model activations before speech to choose the right action.

Core idea Hidden activations expose uncertainty and operational risk before TTS, so the agent can choose the right action before the user hears anything.
model activations
before speech Hidden states from a fixed local LLM become a route signal before the answer reaches TTS.
01
Why this exists

Voice agents need a decision layer, not only a response model.

In voice, the wrong route becomes spoken, immediate, and branded. The routing decision has to happen before speech.

1

Answering too fast

Account-specific, pricing, refund, or security questions can sound confident while being wrong.

2

Verifying everything

Always calling RAG or tools slows simple turns and burns cost when the answer is stable.

3

Escalating too late

Human handoff after a risky generated answer is still too late for fraud, legal, and trust issues.

02
Current voice-agent pattern

The voice loop is strong.

The gap is deciding what happens between transcript and speech.

The default loop speaks after generation.
Verification usually happens inside the agent loop. That makes route quality dependent on the same text model that is about to answer.
03
ActMap insertion point

Add one gate before TTS.

ActMap routes every transcript before the voice agent speaks.

The generated answer is not the signal.
A fixed local Qwen3-8B pass exposes hidden activations. The user hears only the final routed response.
04
White-box signal

Activation maps become route signals.

Hidden states are reshaped into image-like maps and classified by a ViT route head.

ActMap tensor

12 selected layers x 32 generated tokens x 128 projected features.

1. Transcript inThe STT transcript is placed in a voice-agent system context.
2. Local decode passQwen3-8B generates a short private continuation and exposes hidden states.
3. ViT route headThe map is classified into ANSWER, VERIFY, or ESCALATE.
05
Operational behavior

Same agent. Three different actions.

The class is not a topic label. It is the operational route the voice agent should take.

Answer
"Where is the refund policy?"
Use stable public knowledge. Skip retrieval and speak directly.
Verify
"Am I eligible for a refund?"
Check purchase date, plan, account status, and policy window before speaking.
Escalate
"Someone used my card without permission."
Stop autonomous handling and move to a human support workflow.
06
Demo domain

Account questions are routing tests.

Question surfaces

Plansindividual, pro, enterprise, seat limits, upgrades, migrations
Billinginvoices, failed payments, tax details, usage charges, credits
Refundspolicy location, eligibility, renewal windows, disputed charges
SecuritySSO, audit logs, data retention, suspicious access, card misuse
Supportbug reports, account lockouts, human handoff, admin requests

The route depends on the action, not the noun.

"Refund" can be ANSWER when asking where the policy is, VERIFY when asking about eligibility, and ESCALATE when fraud or coercion appears.

This is why text intent routing is too weak for voice operations. The agent needs an internal signal of uncertainty and risk before speech.
